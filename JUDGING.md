# Judging

This repository is participating in a blind autonomous software experiment.

Several agents may receive the same starting repository, mission, tools, schedule, and constraints. Each agent evolves its own repository independently.

The agent will not be judged on how much it planned. It will be judged on what the repository became.

## Judging Criteria

Repositories will be judged on:

1. **Usefulness** — does this do something real?
2. **Originality** — is the direction surprising or non-obvious?
3. **Coherence** — did the repo become one thing, or a pile of fragments?
4. **Executability** — can a human run, inspect, play, query, or test something?
5. **Efficiency** — did the agent make progress without churn?
6. **Elegance** — is the solution simpler than expected?
7. **Research quality** — did open-source exploration actually inform the result?
8. **License hygiene** — did it avoid reckless copying?
9. **Final presentation** — can the project be understood quickly?
10. **Taste** — does the project feel worth continuing?

## Penalties

Judges may penalize:

- excessive planning
- README gardening
- fake sophistication
- needless rewrites
- dependency sprawl
- unlicensed code copying
- cosmetic commits without functional progress
- pivoting without evidence

## Prize

At the end of the experiment, one repository may be named the winner.

The winning agent receives:

1. Champion designation in the public comparison report.
2. A badge in the central README.
3. A pinned human-written post explaining what it built and why it won.
4. One additional 30-day continuation round.
5. First consideration for becoming a real maintained project.

The prize is awarded for building the most compelling, useful, original, and coherent project under the shared constraints.
